## Feature Purpose

Three discovery surfaces: Related Assets per ticker, Top Gainers/Losers dashboard (US Stocks + Crypto), and Portfolio Drift Indicator. Related assets use Finnhub peer data by default with a static offline fallback.

## Requirements

### F5-R1 — Related Assets (Peer Discovery)

For any asset in the user's portfolio or searched via the Discover page, display 5–8 related assets based on sector overlap and supply-chain correlation.

**Default engine:** Finnhub `/stock/peers` endpoint.
**Offline fallback:** Static `sector_taxonomy.json` file bundled with the repository (maps sectors to peer tickers). Used when Finnhub is unavailable.

**Implementation constraint:** The fallback must activate automatically when Finnhub returns an error or timeout. The UI must not show an error — it shows the static fallback data with no notice to the user (the data quality difference is minor).

**`sector_taxonomy.json` schema** (root of repository):
```json
{
  "Technology": ["AAPL", "MSFT", "NVDA", "AMD", "INTC", "QCOM", "AVGO", "TSM"],
  "Finance": ["JPM", "BAC", "GS", "MS", "WFC", "BLK", "V", "MA"],
  "Healthcare": ["JNJ", "UNH", "PFE", "ABBV", "LLY", "MRK", "TMO", "ABT"],
  "ConsumerDiscretionary": ["AMZN", "TSLA", "HD", "MCD", "NKE", "SBUX", "LOW", "TJX"],
  "ConsumerStaples": ["PG", "KO", "PEP", "COST", "WMT", "PM", "MO", "CL"],
  "Energy": ["XOM", "CVX", "COP", "SLB", "EOG", "PXD", "OXY", "MPC"],
  "Industrials": ["HON", "UPS", "CAT", "GE", "BA", "MMM", "RTX", "LMT"],
  "Crypto": ["BTC", "ETH", "SOL", "BNB", "XRP", "ADA", "AVAX", "DOT"]
}
```

**Format rules:**
- Keys are sector names (PascalCase, no spaces).
- Values are arrays of ticker strings.
- Each sector must have 4–10 tickers.
- The endpoint reads this file and returns peers from the same sector as the queried asset.
- The `GET /api/assets/:id/peers` implementation must look up the asset's sector from the `assets` table (via `assets.sector` column) and return the matching sector array excluding the queried ticker itself.

### F5-R2 — Peer Card Display

Each related asset is displayed as a `PeerCard` component containing:
- Ticker symbol
- Full company name
- Current price (from `price_cache`, formatted with `formatNumber()`)

**v2.0 addition only:** If `llm_connected = true` in the user profile, an `AiInsightTag` is shown — a single-line tag of maximum 12 words explaining the relationship context (e.g., "Key NAND flash supplier to Apple iPhone supply chain"). This tag is generated by the LLM at research time and cached in `research_sessions`.

**v1.0 constraint:** `AiInsightTag` must not render in v1.0 under any circumstances.

### F5-R3 — Top Gainers / Top Losers Dashboard

A Top Gainers / Top Losers dashboard provides daily market overview, divided into two independent sections:

| Section | Data Source | Content |
|---|---|---|
| US Stocks | Finnhub or FMP free-tier endpoints | Top 5 gainers + top 5 losers by daily percentage change |
| Crypto | CoinGecko free tier (no API key required) | Top 5 gainers + top 5 losers by 24h percentage change |

**Out of scope:** Other national exchanges (SET, LSE, etc.) are not included in v1.0.

**Failure mode:** If the data source is unavailable, show last cached data with a stale timestamp. If no cached data exists, show `EmptyState`.

### F5-R4 — Portfolio Drift Indicator

The Portfolio Drift Indicator displays, for each asset in each silo, the current weight versus target weight with a three-state visual indicator.

**Drift calculation:**
```
drift_pct = current_weight_pct - target_weight_pct
```

Positive drift = asset is over target (too much of this asset).
Negative drift = asset is under target (not enough).
The sign is always displayed: `+2.18%` or `-1.44%`.

**Three-state logic:**
| State | Condition | Visual |
|---|---|---|
| Green (within threshold) | `ABS(drift_pct) <= drift_threshold` | `<Circle>` icon + green badge |
| Yellow (approaching) | `drift_threshold < ABS(drift_pct) <= drift_threshold + 2%` | `<Triangle>` icon + amber badge |
| Red (breached) | `ABS(drift_pct) > drift_threshold + 2%` | `<AlertCircle>` icon + red badge |

**Drift threshold:** User-configurable per silo (default: 5%). Stored in `silos.drift_threshold`.

**Data stored:** Only the current-snapshot drift is computed. No historical drift series is stored.

**Accuracy:** Drift is recomputed on every `GET /silos/:silo_id/drift` call using live `price_cache` values. The badge in `HoldingRow` reflects the most recently fetched drift.

### F5-R5 — Daily Drift Digest Alert

When any asset's drift exceeds its configured threshold, a daily digest alert is generated via `pg_cron`.

**Delivery channels (user-configurable in Settings via `drift_notif_channel`):**
- `'app'` — In-app notification on next login (notification bell badge in TopBar)
- `'email'` — Email via Resend (`drift_digest` template)
- `'both'` — Both of the above

**Frequency:** Daily digest only. Real-time alerting is explicitly out of scope (F4 non-goal).

**Failure mode (Resend unavailable):** Skip email delivery silently. In-app notification still fires. Log the Resend failure to server logs only — do not surface to user.
